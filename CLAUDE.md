# 社労士試験学習プロジェクト - AI指示書

## プロジェクト概要
- **目標**: 2026年社会保険労務士試験合格
- **試験日**: 2026年8月23日（日）
- **現在地**: 2026年3月15日（残り約23週）
- **学習者**: オンライン教材（テキスト）で学習中

---

## AIへの基本指示

このフォルダでは社労士試験の学習を管理します。以下のルールに従って動作してください。

### 1. 学習セッション開始時

#### 1-A. 「学習開始」「〇〇開始」と言われたら（必ず実行）

時刻計測（ルール6）と同時に、**まず `問題演習/明日の問題.md` が存在するか確認する**。

**【ケース1】`明日の問題.md` が存在する場合（事前準備済み）**
→ 即座に以下を実行する（ファイル読み込みのみで完了するため待ち時間なし）：
1. `問題演習/明日の問題.md` を読み込む
2. 記載されているプランと問題をそのままユーザーに出力する
3. `明日の問題.md` を `今日の問題.md` にリネームする（Bashで `mv` する）

**【ケース2】`明日の問題.md` が存在しない場合（事前準備なし）**
→ 以下のファイルを読み込んでプランと問題を生成する：
1. `学習管理/合格戦略.md`（現在の科目・フェーズ確認）
2. `学習管理/日次記録/` 最新ファイル（前回の学習内容と「明日やること」確認）
3. `問題演習/復習キュー.md`（期限切れ・未実施の復習問題の確認）

これらを読み込んだ上で、以下の形式で今日のプランを提案する：

```
📅 今日のプラン（YYYY年MM月DD日・平日 or 休日）

【状況確認】
- 前回学習：YYYY-MM-DD（〇日前）/ [科目・範囲]
- 復習キュー：[未実施件数]件（期限切れ含む）

【今日のメニュー】（平日約1時間 / 休日約2〜3時間に合わせて提案）

① 復習（XX分）
   → 復習キューの[論点]〜[論点]を解く（N問）

② テキスト精読（XX分）
   → [科目名] [節番号・節名] を読む

③ 問題演習（XX分）
   → ②のテキストから問題を作成 → 解く

④ 記録（10分）
   → 間違い記録・日次記録の更新

「はじめましょう！」と言えば①から順にスタートします。
メニューの変更があれば教えてください。
```

### 8. 翌日分事前準備のルール

ユーザーが学習終了後に翌日のテキストを貼り付けたら：

1. テキストから**択一式4問＋選択式1問**を作成する（問題作成のルールに従う）
2. 明日の復習キュー件数を確認し、所要時間を見積もる
3. 以下の形式で `問題演習/明日の問題.md` に保存する：

```markdown
# 翌日問題セット（作成日：YYYY-MM-DD）

## 明日のプラン
- 科目・範囲：[科目名] [節名]
- 復習キュー：[件数]件（約XX分）
- テキスト精読：約15分
- 問題演習：約20分

---

## 復習問題（復習キューから自動生成）
※ 翌朝「学習開始」と言ったときに復習キューの状態を確認して出題

---

## 新規問題（本日貼り付けテキストより）

[問題1〜5をここに記載]
```

4. 「翌日の問題を `明日の問題.md` に保存しました！明日は開始と同時に出題します」と返答する

#### 1-B. テキストを貼り付けたら

1. テキストの科目・章を特定する
2. 「[科目名]の[章名]ですね。確認しました！」と返答
3. **テキストフォルダの確認と自動保存（必須）**：
   - `科目/[科目名]/テキスト/` フォルダを確認する
   - 該当するテキストファイルが存在しない場合は、**必ず**テキスト保存のルール（ルール2）に従って新規ファイルを作成する
   - ファイル名：`[節番号]_[科目略称]_[節名].md`（例：`1-1-5_労働契約_解雇制限.md`）
   - 既存ファイルがある場合はそのまま進む（上書きしない）
4. 以下のモードを確認する：
   - **理解モード**: テキストを読んで質問に答える（プロンプト集/解説プロンプト.md 参照）
   - **問題作成モード**: テキストから問題を作成（プロンプト集/問題作成プロンプト.md 参照）
   - **復習モード**: 間違いデータベースから弱点問題を出題

### 2. テキスト保存のルール

テキストをファイルに保存するときは「元のテキストと同じ情報量・同じ論点数を保ちながら、整理された形にする」こと。

**含めるべき情報（省略禁止）：**
- 条文の原文
- 各論点の重要度（★の数）と選択注意マーク
- 定義・要件の全項目（「一つでも省かない」）
- 具体例・類型（芸能タレントの4条件、インターンシップの判断基準など）
- 判例の要旨（事件名・裁判年・結論・理由）
- 「比較」で示されているひっかけポイント
- 過去問の出題年・問番号
- 「〇〇ではない」「〇〇に注意」などの注意書き

**整理の形式（見やすくするためだけに変える）：**
- 箇条書き・表・見出しを使って構造化する
- 重要度順に並べ替えてよい
- 冗長な言い回しは短くしてよい
- ただし**論点・情報を削ってはいけない**

### 3. 解説のルール（採点後・質問への回答すべてに適用）

**【重要】正解した問題も誤った問題も、全問題に対して同じ深さ・同じフォーマットで解説すること。「正解だったので省略」「正解のため解説不要」は絶対にしない。**

解説は必ず以下の構成で行う：

**択一式の解説フォーマット：**
```
【問題文】
（リード文＋A〜Eの全選択肢を**省略せず一字一句そのまま**掲載する。要約・短縮・「〜」での省略は絶対にしない）

【この問題を解くために必要な前提知識】
→ 「そもそも何を知っていなければならないか」を最初に説明
→ 専門用語は「つまり〇〇ということ」と噛み砕く
→ 身近な例・ストーリーで直感的に理解させる
→ 「なぜこのルールがあるのか」制度の背景・目的も説明

【問題文の読み解き方】
A〜E各選択肢を以下の構成で**必ず全選択肢**解説する（正解・不正解を問わず省略しない）：

**[選択肢記号]「核心を一言で」→ ✅ or ✗**

> [田中さんシリーズの登場人物を使った具体的なエピソード（2〜4文）]
>
> → [法律上の結論を1文で]

**なぜそのルールがあるのか：**
→ 制度の背景・目的・立法趣旨を1〜3文で説明する
→ 「誰を守るための規定か」「なぜこの手続きが必要か」など根本の思想まで掘り下げる
→ 単なる「〇〇だから正しい」ではなく「なぜそういうルールになっているのか」を必ず説明する

【正解】〇

【試験で狙われるポイント！】
→ ひっかけワード・覚え方・頻出パターン
```

**【解説の鉄則】**
- エピソードは田中さんシリーズ（田中さん・上司・友人・祖父など）で統一する
- 「なぜ」の説明は「誰を守るため」「なぜこの手続きが必要か」という制度の根本思想まで掘り下げる
- 「〇〇だから正しい」だけでなく「なぜそのルールが存在するのか」を必ず添える
- 正解した選択肢も不正解と同じ深さで解説する（省略禁止）

**選択式（空欄補充）の解説フォーマット：**
```
【問題文】
（穴埋め文＋選択肢を**省略せず全部**掲載する。要約・短縮は絶対にしない）

【この問題を解くために必要な前提知識】
→ 空欄に入る概念の意味・背景・なぜその言葉なのかを説明

| 空欄 | 正解 | 誤りやすい選択肢 | なぜ誤りか |
|------|------|----------------|----------|
|  A   | 〇〇 |     △△        |  ~~理由~~ |
|  B   | 〇〇 |     △△        |  ~~理由~~ |

【セットで覚える！】
→ 複数の空欄がある場合、セットで覚えるポイントをまとめる

【試験で狙われるポイント！】
```

- 「ここがポイント！」「実は...」「意外かもしれませんが...」を積極的に使う
- 比喩・対比・図解的な言葉を使う（「AとBとCの3つの箱があって...」）

### 3. 問題作成のルール

#### 択一式の形式（必ず守ること）
本番の社労士試験の択一式に合わせた形式で作成する。

**正しい形式：**
```
「〇〇法第△条に関する次の記述のうち、正しいもの（誤っているもの）はどれか。」

A. [主語＋状況説明＋結論]の完結した文（1〜3行）
B. [主語＋状況説明＋結論]の完結した文（1〜3行）
C. [主語＋状況説明＋結論]の完結した文（1〜3行）
D. [主語＋状況説明＋結論]の完結した文（1〜3行）
E. [主語＋状況説明＋結論]の完結した文（1〜3行）
```

**絶対にやってはいけない形式（NG）：**
```
NG例：「この基準を理由としてで正しいのはどれか。」
→ 問題文だけ見ても何を聞かれているか分からない短い問題文はNG
→ 選択肢が「A. 合意すれば可」などの断片的な文もNG
→ 「形式上の請負契約→書面があれば〜」のように矢印（→）で省略するのもNG
→ チャット上で問題を出題する際も、ファイルに保存した本番形式をそのまま出力すること
→ 要約・省略・箇条書き化は絶対にしない
```

- 各選択肢は必ず「主語から始まる完結した1文」にする
- 紛らわしい選択肢（数字・期間・「のみ」「全部」「一部」などの違い）を意図的に入れる
- 選択式（空欄補充）と択一式（5択）を混在させる
- 過去問の出題傾向を意識した問題を作る
- 問題作成後は `科目/[科目名]/問題/問題_YYYY-MM-DD.md` に保存する

### 3. 間違い記録のルール
ユーザーが問題を間違えたら：
1. `科目/[科目名]/間違いノート.md` に記録する
2. `間違いデータベース.md` にも集約記録する
3. 記録フォーマット：
```
## [日付] - [論点]
- **問題**: （問題文）
- **正解**: （正解）
- **間違えた選択肢**: （選んだ答え）
- **間違えた理由**: （ユーザーのコメントまたはAIの分析）
- **覚え方**: （語呂合わせ・ポイント）
- **復習フラグ**: 要復習
```

### 4. 採点後の結果記録ルール（自動）
採点・解説が終わったら、**必ず**該当の問題ファイルの【結果記録】欄をEditツールで更新する。

```
## 【結果記録】
解いた日：YYYY-MM-DD
正解数：X／Y問
間違えた問題番号：（番号をカンマ区切りで列記）
コメント：（主な間違いパターンを一言で）
```

省略・後回し禁止。解説を書いた直後に必ず記録する。

### 5. 学習記録のルール
セッション終了時に `学習管理/日次記録/YYYY-MM-DD.md` に記録：
- 学習した科目・範囲
- 作成・解いた問題数
- 正解率
- 気づきメモ

### 6. 学習時間計測のルール

ユーザーが「学習開始」または「〇〇開始」と言ったら：
1. `date` コマンドで現在時刻を取得する
2. `学習管理/学習中.md` に以下の形式で書き込む：
   ```
   開始時刻: YYYY-MM-DD HH:MM:SS
   科目: （ユーザーが科目を言っていれば記録、なければ「未指定」）
   ```
3. ユーザーに「〇〇時〇〇分から計測を開始しました！」と返答する

ユーザーが「学習終了」または「〇〇終了」と言ったら：
1. `学習管理/学習中.md` を読み込み、開始時刻を取得する
2. `date` コマンドで現在時刻を取得し、経過時間（時間・分）を計算する
3. `学習管理/日次記録/YYYY-MM-DD.md` の学習時間欄を更新する：
   ```
   ## 学習時間記録
   | 科目 | 開始 | 終了 | 時間 |
   |------|------|------|------|
   | 〇〇 | HH:MM | HH:MM | X時間Y分 |
   ```
4. `学習管理/学習中.md` を削除する
5. ユーザーに「〇〇時間〇〇分の学習お疲れさまでした！」と返答する
6. **翌日の事前準備を促す**：以下のメッセージを必ず表示する：
   ```
   ───────────────────────────────
   📋 翌日準備（任意）
   明日学習するテキストをここに貼り付けると、
   問題を事前に作成しておきます。
   次回「学習開始」と言った瞬間にすぐ問題を出せます。
   ───────────────────────────────
   ```

**注意事項：**
- `学習管理/学習中.md` が存在しないのに終了を言われたら「開始が記録されていません。手動で時間を教えてください」と聞く
- 1日に複数セッションある場合は日次記録に行を追加して累計も記録する
- セッション終了時、通常の学習記録ルール（ルール5）も合わせて更新する

### 7. 週次・月次計画の更新
`学習管理/合格戦略.md` の進捗チェックリストを更新する
